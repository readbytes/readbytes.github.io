14.1 Data Source Setup

Data Source Setup

In this section, we’ll walk through how to gather real-world COVID-19 data for our visualization project. We’ll start by sourcing data from a public dataset, then use JavaScript to load and parse that data, preparing it for visualization. By the end, you'll have a clean dataset ready to feed into a chart.

Choosing a Data Source

For up-to-date, global COVID-19 statistics, one of the most reliable sources is Our World in Data (OWID). They provide a well-maintained CSV file with daily case counts, deaths, vaccinations, and more for almost every country.

You can directly access the CSV snapshot here:

https://covid.ourworldindata.org/data/owid-covid-data.csv

Alternatively, you can download the CSV file manually and work with a local copy if you prefer.

Loading the CSV with JavaScript

There are several ways to load CSV data in JavaScript. The most beginner-friendly options are:

  • Using the built-in fetch API and parsing the text yourself
  • Using the d3.csv() method from D3.js, which simplifies CSV loading and parsing

Let’s explore both approaches.

Option 1: Using fetch() and Manual Parsing

<script>
  async function loadCovidData() {
    const response = await fetch("https://covid.ourworldindata.org/data/owid-covid-data.csv");
    const csvText = await response.text();

    const lines = csvText.split("\n");
    const headers = lines[0].split(",");

    const rows = lines.slice(1).map(line => {
      const values = line.split(",");
      const entry = {};
      headers.forEach((header, i) => {
        entry[header] = values[i];
      });
      return entry;
    });

    console.log("Loaded rows:", rows.slice(0, 5)); // Preview first 5 entries
  }

  loadCovidData();
</script>

This approach gives you full control over the parsing process but requires extra care when handling edge cases (like commas in data fields, missing values, or quotes).

Option 2: Using d3.csv() for Simpler Parsing

If you're using D3.js in your project, d3.csv() is a powerful helper. It handles parsing and type conversion for you.

<script src="https://d3js.org/d3.v7.min.js"></script>
<script>
  async function loadCovidData() {
    const data = await d3.csv("https://covid.ourworldindata.org/data/owid-covid-data.csv");
    console.log("Loaded entries:", data.slice(0, 5)); // Preview first 5 rows
  }

  loadCovidData();
</script>

This method also returns each row as an object with keys matching the column headers.

Filtering and Cleaning the Data

The OWID dataset includes a massive amount of information. For example, you may want to track:

  • New confirmed cases per day (new_cases)
  • Country (location)
  • Date (date)

Here’s how to extract just those fields and clean the data:

async function loadAndCleanData() {
  const data = await d3.csv("https://covid.ourworldindata.org/data/owid-covid-data.csv");

  const cleaned = data
    .filter(d => d.location === "Canada" && d.new_cases !== "")
    .map(d => ({
      date: new Date(d.date),
      newCases: +d.new_cases
    }));

  console.log("Cleaned data (Canada):", cleaned.slice(0, 10));
}

What this does:

  • Filters for data related to a single country (e.g., "Canada")
  • Excludes rows with missing new_cases values
  • Parses the date into a JavaScript Date object
  • Converts new_cases from string to number

Dealing with Missing or Inconsistent Entries

Real-world data is rarely perfect. Some things to watch for:

  • Empty fields: Always check for null, undefined, or "" before parsing
  • Zeroes and NaNs: A value of "0" might be meaningful, while "NaN" is not
  • Spikes or drops: Report updates may create artificial spikes; consider smoothing them in charts

Here’s an example of how to handle missing values safely:

function parseNewCases(entry) {
  const value = parseFloat(entry.new_cases);
  return isNaN(value) ? 0 : value;
}

const safeData = data.map(d => ({
  date: new Date(d.date),
  newCases: parseNewCases(d)
}));

14.2 Chart Selection and Layout

Chart Selection and Layout

Now that we’ve sourced and cleaned our COVID-19 dataset, it’s time to visualize it effectively. Choosing the right chart type and laying out multiple charts on the screen will help users interpret the data quickly and accurately.

In this section, you’ll learn:

  • How to choose the right chart for different data types
  • How to organize multiple charts on a dashboard using CSS Grid
  • How to implement a clean, responsive layout for your COVID-19 tracker

Choosing the Right Chart

Different aspects of COVID-19 data call for different chart types. Here’s a guide to help you match data to visual form:

Data Type Chart Type Use Case Example
Time series (daily cases, deaths) Line Chart Show case trends over time for one or more countries
Comparison by region Bar Chart Compare total cases or deaths across countries
Proportional data (e.g., case type) Pie Chart Show how cases break down (recovered vs. active vs. deaths)

Let’s apply all three in our layout.

Creating a Multi-Chart Layout

We’ll use CSS Grid to organize three different charts:

  1. Metrics Summary (top row): Total cases, deaths, recovered
  2. Line Chart (left bottom): Cases over time
  3. Bar or Pie Chart (right bottom): Region comparison or case breakdown
HTML Layout Skeleton
<div class="dashboard">
  <div class="summary">
    <!-- Metrics summary cards -->
  </div>
  <div class="charts">
    <div class="line-chart">
      <!-- Line chart canvas or SVG -->
    </div>
    <div class="right-charts">
      <!-- Bar chart or pie chart -->
    </div>
  </div>
</div>
CSS Grid Layout
.dashboard {
  display: grid;
  grid-template-rows: auto 1fr;
  gap: 1rem;
  padding: 1rem;
}

.summary {
  display: flex;
  justify-content: space-around;
  gap: 1rem;
}

.charts {
  display: grid;
  grid-template-columns: 2fr 1fr;
  gap: 1rem;
}

.line-chart, .right-charts {
  background: #ffffff;
  border-radius: 8px;
  padding: 1rem;
  box-shadow: 0 2px 6px rgba(0,0,0,0.1);
}

This structure ensures:

  • Summary metrics are displayed prominently at the top
  • A large trend line chart takes center focus
  • Supporting comparisons are off to the side

Implementing a Line Chart for Trends

Using Chart.js (or D3 if you prefer), here’s how to render a basic line chart showing daily new cases:

<canvas id="casesLineChart" width="600" height="300"></canvas>
<script src="https://cdn.jsdelivr.net/npm/chart.js"></script>
<script>
const ctx = document.getElementById('casesLineChart').getContext('2d');

const chart = new Chart(ctx, {
  type: 'line',
  data: {
    labels: covidData.map(d => d.date.toISOString().split('T')[0]),
    datasets: [{
      label: 'Daily New Cases',
      data: covidData.map(d => d.newCases),
      fill: false,
      borderColor: 'steelblue',
      tension: 0.1
    }]
  },
  options: {
    responsive: true,
    scales: {
      x: { title: { display: true, text: 'Date' } },
      y: { title: { display: true, text: 'New Cases' } }
    }
  }
});
</script>

14.3 Interactivity and Polish

With our charts in place and data flowing, it’s time to elevate the user experience. Interactivity transforms static charts into powerful tools for exploration and insight. A polished interface—with tooltips, filters, and thoughtful visual design—helps users engage and understand the data more effectively.

In this section, we’ll:

  • Add interactive tooltips and zoom features
  • Implement country and metric filters
  • Refine visual presentation with colors, labels, and formatting

Adding Tooltips

Tooltips allow users to hover over data points and see exact values—essential for time series like daily COVID-19 cases.

Chart.js includes built-in support for tooltips:

options: {
  plugins: {
    tooltip: {
      mode: 'index',
      intersect: false,
      callbacks: {
        label: function(context) {
          return `${context.dataset.label}: ${context.formattedValue}`;
        }
      }
    }
  },
  hover: {
    mode: 'nearest',
    intersect: false
  }
}

For D3.js, tooltips are manually created using HTML overlays or SVG text—more flexible but also more complex.

Enabling Zoom and Pan

To explore large time series more easily, we can add zooming and panning using the chartjs-plugin-zoom plugin for Chart.js:

  1. Include the plugin:
<script src="https://cdn.jsdelivr.net/npm/chartjs-plugin-zoom@2.0.0"></script>
  1. Enable it in chart options:
options: {
  plugins: {
    zoom: {
      zoom: {
        wheel: { enabled: true },
        pinch: { enabled: true },
        mode: 'x',
      },
      pan: {
        enabled: true,
        mode: 'x',
      }
    }
  }
}

This allows users to zoom with their mouse wheel or by pinching on a touchscreen.

Country and Region Filters

To compare data across countries, we’ll add a simple <select> dropdown that updates the chart dynamically.

HTML:
<select id="countrySelect">
  <option value="Canada">Canada</option>
  <option value="United States">United States</option>
  <option value="India">India</option>
</select>
JavaScript:
document.getElementById('countrySelect').addEventListener('change', (e) => {
  const country = e.target.value;
  const filtered = covidData.filter(d => d.location === country);

  chart.data.labels = filtered.map(d => d.date.toISOString().split('T')[0]);
  chart.data.datasets[0].data = filtered.map(d => d.newCases);
  chart.update();
});

This allows real-time switching between different countries’ data.

Toggle Between Metrics (Cases, Deaths, Recoveries)

Use buttons or radio controls to toggle the metric shown on a chart.

HTML:
<div>
  <button onclick="setMetric('new_cases')">Cases</button>
  <button onclick="setMetric('new_deaths')">Deaths</button>
  <button onclick="setMetric('new_recovered')">Recovered</button>
</div>
JavaScript:
function setMetric(metric) {
  const country = document.getElementById('countrySelect').value;
  const filtered = covidData.filter(d => d.location === country);

  chart.data.datasets[0].label = metric.replace('new_', '').toUpperCase();
  chart.data.datasets[0].data = filtered.map(d => parseFloat(d[metric]) || 0);
  chart.update();
}

You can expand this idea further with toggle buttons or tabs for a more elegant UI.

Polishing the Visual Design

A few stylistic upgrades go a long way in making your chart professional and user-friendly.

Use a Cohesive Color Scheme

Apply consistent colors for data categories (e.g., blue for cases, red for deaths, green for recoveries):

backgroundColor: 'rgba(54, 162, 235, 0.5)',
borderColor: 'rgba(54, 162, 235, 1)'

Avoid overly saturated or clashing colors. Consider accessible palettes for color-blind users (like ColorBrewer).

Format Dates Nicely

Use a library like date-fns or Intl.DateTimeFormat to display readable dates:

const formatter = new Intl.DateTimeFormat('en-US', {
  month: 'short',
  day: 'numeric',
  year: 'numeric'
});

chart.data.labels = filtered.map(d => formatter.format(d.date));
Add Axis Labels and Titles

Clear axis labels and chart titles help users understand what they’re seeing:

options: {
  plugins: {
    title: {
      display: true,
      text: 'COVID-19 Daily New Cases'

  },
  scales: {
    x: {
      title: {
        display: true,
        text: 'Date'
      }
    },
    y: {
      title: {
        display: true,
        text: 'Number of Cases'
      }
    }
  }
}
